FillPro Support

Help for profile setup, on-page filling, smart rules, local file uploads, privacy questions, and known site limitations.

Fast Start

  1. Create a profile in the FillPro popup.
  2. Add custom fields, smart rules, or local uploads if needed.
  3. Open a normal web form and click Fill Page, use the on-page picker, or press Ctrl+Shift+F.

Best Use Cases

  • Job applications and candidate forms
  • Account creation and onboarding flows
  • Checkout contact and shipping sections
  • Lead capture, demo request, and contact forms
  • Vendor, partner, and client intake forms

Privacy Promise

  • Profile data stays in your browser.
  • No account is required for free use.
  • No analytics or background form-data collection.
  • Passwords, payment fields, SSNs, tax IDs, and one-time codes are skipped by design.

Using Smart Rules

Smart rules help when a site uses awkward field names or labels that do not match normal profile keys.

  • Use plain text for contains matching, for example work email.
  • Use JavaScript regex literal syntax for harder cases, like /best.*contact/i.
  • Use @field references to reuse existing profile values, like @email or @company_size.

Using Local Uploads

Attach a resume, CV, cover letter, or similar file inside a profile. FillPro stores a local encoded copy in extension storage and uses it only when you choose to fill a matching upload field.

  • Supported formats: PDF, DOC, DOCX, TXT, RTF, ODT, and Pages
  • Current per-file limit: 2 MB
  • Use a target pattern like resume, cv, or /cover.?letter/i.

Known Limits

  • Sites behind CAPTCHA, bot checks, or human verification must be cleared manually first.
  • Cross-origin embedded forms can remain unreachable under Chrome's privacy-first activeTab model.
  • Browser-native PDF tabs are not fillable. Compatible PDF.js-based HTML viewers are the supported path.
  • Payment cards, CVC, passwords, SSNs, tax IDs, and one-time codes are intentionally skipped.
  • Some sites intentionally block automation. In those cases, FillPro should report the blocker honestly instead of faking success.

Troubleshooting

  • If nothing fills, wait until the real form is visible, then try again.
  • If the wrong field was matched, add a smart rule for that site-specific label.
  • If an upload field is missed, add a more explicit file-target pattern like candidate resume or /curriculum/i.
  • If a page is dynamic, try the on-page picker after the form has fully appeared.
